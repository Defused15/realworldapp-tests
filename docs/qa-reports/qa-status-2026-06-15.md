## The business case — why this is worth the money

**1. Cost of a defect escalates 1× → 10× → 100×.** A bug caught in development
costs ~1×; in QA ~10×; in production ~100× (rework + incident response + support +
churn + reputation). This suite has already caught **8+ issues before release** —
each one is that 100× production cost avoided. The IDOR flaw alone, if shipped,
is a potential data-breach-class incident.

**2. Automation ROI (conservative range).**

- ~364 automated functional checks run in **~35 seconds**.
- Manual equivalent ≈ 306 UI/API checks × ~3 min each ≈ **~15 hours per full
  regression**.
- At ~12–52 releases/year and a ~$30–60/hr QA rate, that is roughly
  **$5,500 – $48,000/year of manual testing labor avoided** (mid-case ~$15k).
- Build cost of the suite (~80–120 hrs once) breaks even after **~6–8 releases** —
  typically the first quarter.

**3. Faster feedback = faster, safer delivery.** Gated CI returns pass/fail in
minutes instead of days of manual regression, enabling more frequent releases with
less risk (DORA: lower lead time, lower change-failure rate).

**4. Coverage as insurance.** The critical money-paths (auth, transaction feed)
have a regression safety net; the CI gates stop a regression from reaching users.

**5. Automating the metrics themselves.** This report is generated from live data,
not a hand-built spreadsheet — so quality status is always current, auditable, and
trend-able. The client gets **continuous assurance**, not a one-off snapshot.
